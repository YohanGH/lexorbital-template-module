# Code of Conduct – LexOrbital

LexOrbital is a modular, compliance-driven architecture ecosystem built on principles of  
simplicity, security, and clear structure.  
The quality of the project depends on both good engineering practices and respectful collaboration.

This Code of Conduct defines expectations for all contributors.

---

## 1. Purpose

This document aims to:

- ensure a **respectful**, **professional**, and **inclusive** environment;
- encourage high-quality technical contributions;
- maintain a governance aligned with the spirit of the project  
  (simplicity, consistency, compliance, modular architecture).

The community must remain a safe space for everyone.

---

## 2. Expected Behavior

Contributors agree to:

- show respect and consideration toward others;
- provide constructive and honest feedback;
- ask questions openly and respond with patience;
- accept technical criticism without hostility;
- follow project conventions (style, structure, security, documentation);
- act in ways that improve the overall quality of the LexOrbital ecosystem.

---

## 3. Unacceptable Behavior

The following behaviors will **not be tolerated**:

- harassment, intimidation, or personal attacks;
- discriminatory or offensive comments regarding gender, origin, religion, disability,  
  sexual orientation, or any personal characteristic;
- trolling, deliberate disruption, or dismissive attitudes toward others’ work;
- publishing private information without explicit permission;
- using GitHub tools to spread inappropriate content;
- malicious or intentionally harmful contributions (e.g., deliberate vulnerabilities).

Maintainers may take any action deemed necessary,  
including issue locking, temporary bans, or permanent removal from the repository.

---

## 4. Compliance with LexOrbital Standards

LexOrbital enforces strict technical policies:

- **Conventional Commits**
- **Template-based module structure**
- **Security by design**
- **Standardized TypeScript/React/Node conventions**
- **Mandatory minimal documentation**
- **Avoidance of unnecessary dependencies**

Contributing implies respecting these principles at all times.

---

## 5. Reporting Misconduct

If you witness or experience inappropriate behavior:

- contact the project maintainers privately (email available in the README);
- provide a clear description of the incident (context, screenshots, links to issues).

All reports will be handled **confidentially**.

---

## 6. Maintainers’ Responsibilities

Maintainers are responsible for:

- clarifying the rules of this Code of Conduct;
- taking corrective actions when needed;
- preserving a constructive culture within the LexOrbital community.

They may sanction any behavior violating this Code  
(at their discretion, depending on severity).

---

## 7. Commitment

By contributing to LexOrbital, you agree to uphold this Code of Conduct and  
help foster an engineering environment that is:

**calm, respectful, structured, demanding, and aligned with the ambition of the project.**

Thank you for helping make LexOrbital a place where code, ideas, and people  
coexist in a healthy and creative orbit.
